# Research: лендинг-портфолио цифрового специалиста

Исследование для проекта — лендинг-портфолио специалиста, создающего сайты, приложения и ИИ-агентов.
Дата: 2026-07-01

---

## 1. Конкуренты и примеры

Изучены awwwards.com, dribbble.com, behance.net и общий веб-поиск по портфолио-лендингам разработчиков/дизайнеров/AI-инженеров. Найдено 9 примеров:

### 1. Brittany Chiang — brittanychiang.com
- **Hook:** мгновенная ясность профиля frontend-инженера (Klaviyo, ex-Apple), фирменная анимация курсора-«свечения», пасхалка (TARDIS).
- **Секции:** Hero → липкая боковая навигация (About / Experience / Projects) → About → Experience (таймлайн 2015–2024 со стеком технологий) → Projects → Writing → Footer.
- **Стиль:** тёмная тема, строгий минимализм, неоново-зелёный акцент (#64ffda) на тёмно-синем фоне.
- **Шрифты:** Inter для текста, моно-шрифт (SF Mono/Calibre) для тегов и кода. Next.js + Tailwind.

### 2. Sharlee (Charles Bruyerre) — itssharl.ee
- **Hook:** дружелюбный никнейм-бренд + анимированные «materializing shapes» на полноэкранном холсте, переключатель темы.
- **Секции:** Hero-приветствие → Work → About → Contact.
- **Стиль:** минимализм, светлая/тёмная тема на выбор, геометрические анимированные формы как визуальный якорь.
- **Шрифты:** чистый гротеск (Söhne / General Sans).

### 3. Mitchell Hou — mitchellhou.com
- **Hook:** портфолио подано как «живой код» (`import passion`), подчёркивает дуальность designer+developer.
- **Секции:** Hero/Intro (код-блок) → Skills → Experience-таймлайн → манифест из 4 принципов (Clarity, Utility, Empathy, Craft) → CTA/локация.
- **Стиль:** светлый минимализм, монохром с акцентами, «creative code» подача.
- **Шрифты:** гротеск для текста + моно-шрифт (JetBrains Mono/Space Mono) для кодовых блоков.

### 4. Devon Stank — devonstank.com
- **Hook:** видео-фон в hero сразу задаёт кинематографичное, премиальное ощущение.
- **Секции:** hero с видео → about → портфолио работ → контакты.
- **Стиль:** тёмный минимализм, кинематографичный.
- **Шрифты:** крупный гротеск-заголовок (Helvetica Neue-подобный).

### 5. Andrew McCarthy — andrevv.com
- **Hook:** нестандартный «бесконечный скролл» с постоянно меняющимися визуальными элементами — ломает привычный шаблон one-pager.
- **Секции:** непрерывный скролл-нарратив вместо чётких блоков, отдельные точки для проектов.
- **Стиль:** экспериментальный, playful.
- **Шрифты:** моно-гротеск, характерный для «инди-разработчика».

### 6. Diogo Correia — diogotc.com
- **Hook:** частицы-эффект на фоне hero — лёгкое ощущение «технологичности» без перегруза.
- **Секции:** hero с частицами → sticky-навигация → timeline опыта → projects → contact.
- **Стиль:** тёмная тема, минимализм с одной динамической деталью.
- **Шрифты:** системный гротеск (Inter/Roboto-подобный).

### 7. Award-winning developer-сайты (Awwwards, категория Developer, 2025–2026)
Примеры: Wembi (ET Studio), EverSwap (Lusion), Sui (HOLOGRAPHIK) — продуктовые/крипто-лендинги, но задают планку уровня интерактивности.
- **Стиль:** тёмные темы, 3D-объекты, глубокий parallax, кастомные курсоры.
- **Шрифты:** дисплейные (Clash Display / PP Neue Montreal).

### 8. Neodev — шаблон для веб-разработчиков (Astro), AI/tech-направленность
- **Hook:** «Cyberpunk Neon Design» — сразу читается как продукт для tech/AI-специалиста.
- **Секции:** Home (one-page) → About → Services → Portfolio projects → Blog → Contact form → Docs/Legal.
- **Стиль:** тёмная тема, неон-циан + неон-фиолетовый, «cyber-grid», пульсирующие hover-анимации.
- **Шрифты:** моно-инспирированные (JetBrains Mono / Space Grotesk).

### 9. Aura — AI Engineer Portfolio Template ⭐ (наиболее релевантный пример)
- **Hook:** готовый шаблон именно для AI/ML-инженеров — hero с badge «available for work» и статистикой (лет опыта, кол-во моделей/проектов).
- **Секции:** Hero (badge + stats) → Bento-grid проектов → детальные case-study страницы по каждому AI-проекту → стек технологий (Python/PyTorch/LLM-инструменты) → контакты.
- **Стиль:** тёмный режим, минимализм с бенто-сеткой, карточки со glassmorphism.
- **Шрифты:** гротеск (Inter/Geist) + моно-акценты для тех.данных (Space Grotesk, General Sans).

### Вывод по разделу
1. Почти все успешные портфолио — **one-pager со sticky-навигацией**: hero → about/skills → experience/timeline → projects (bento-grid или карточки) → contact/CTA.
2. **Тёмная тема доминирует** среди tech/AI-аудитории, светлая чаще встречается у дизайнеров.
3. Лучшие сайты выбирают **один сильный визуальный приём** (частицы, курсор-свечение, видео-фон, живой код), а не перегружают анимациями.
4. Для AI-тематики специфичен тренд: **неон + глассморфизм + bento-grid карточки проектов** и обязательный блок «стек технологий/статистика» в hero.
5. Шрифтовая пара почти везде одна и та же: чистый гротеск (Inter/General Sans/Space Grotesk) для текста + моно-шрифт (JetBrains Mono/Space Mono) для кода, тегов стека и деталей.

**Рекомендация для проекта:** тёмная тема с неоновым акцентом, bento-grid для кейсов (включая отдельные карточки под ИИ-агентов), timeline опыта, один яркий интерактивный элемент в hero, моно-шрифт для технического стека.

## 2. Структура лучших лендингов

Изучены onepagelove.com, tilda.cc / tildapublishing (блог, «Формула лендинга»), а также involve.me, Wix, Unbounce, Fueler.

### Типовые блоки продающего лендинга (общая теория)
1. **Hero / первый экран** — заголовок с УТП, подзаголовок, визуал, CTA. Задача — за 3 секунды объяснить, что это и для кого.
2. **Боль/проблема клиента** — посетитель узнаёт свою ситуацию.
3. **Услуги/что я делаю** — список предложений через выгоду, а не характеристики.
4. **Кейсы/портфолио** — доказательство навыка: задача → решение → результат.
5. **Процесс работы** — этапы, снимающие тревогу «а как это будет происходить».
6. **Отзывы/доказательства (Trust)** — цитаты клиентов, логотипы, цифры, гарантии.
7. **УТП/почему я** — отличие от конкурентов/агентств.
8. **Тарифы/цены** — снимает вопрос бюджета.
9. **FAQ** — закрытие возражений.
10. **CTA/контакты** — форма или ссылка на мессенджер.
11. **Футер** — соцсети, email, юридическая информация.

По методологии Tilda эти блоки группируются в 5 смысловых зон: **Обложка → Рассказ о проекте → Понятные выгоды → Блоки доверия → Целевое действие**. На OnePageLove в разделе портфолио/фриланс-лендингов чаще встречается укороченная версия (Hero → About → Работы → Услуги → Контакты) — полная 11-блочная схема для личного бренда избыточна.

### Идеальная структура для лендинга-портфолио (7 блоков)

1. **Hero — Оффер**
   Пример заголовка: *«Делаю сайты, приложения и ИИ-агентов, которые приносят деньги, а не просто выглядят красиво»*. Подзаголовок — для кого (стартапы, малый бизнес, эксперты). Личное фото/аватар вместо абстрактной иллюстрации. CTA: «Обсудить проект».

2. **Кому и зачем — короткая боль + позиционирование**
   2-3 буллита о типичных запросах клиента + 1-2 фразы о себе (кто вы, опыт, стек). Без CTA — это связка к следующему блоку.

3. **Услуги / что я делаю**
   3 карточки: «Сайты и лендинги», «Веб/мобильные приложения», «ИИ-агенты и автоматизация» — каждая через выгоду, а не техническое описание.

4. **Кейсы/портфолио — главный блок**
   Формат «Проблема → Решение → Результат» с цифрами. 3-4 лучших кейса, ссылки на живые проекты. Самый длинный и убедительный блок; сюда же можно вставить мини-демо ИИ-агента.

5. **Как я работаю (процесс)**
   4 шага: заявка → бриф и оценка → разработка с промежуточными демо → запуск и поддержка.

6. **Отзывы + УТП «почему я»**
   2-3 отзыва (текст + фото/имя/компания) + короткий блок «почему со мной, а не с агентством»: личное участие, скорость, экспертиза в ИИ-инструментах.

7. **Финальный CTA + контакты + футер**
   Пример: *«Готовы обсудить свой проект?»* Форма из 2-3 полей или кнопка в Telegram/WhatsApp, ответ в течение 24 часов. Ниже — соцсети, email, копирайт.

> Тарифы и отдельный FAQ для фриланс-портфолио чаще избыточны (цена индивидуальна) — можно свернуть в 1-2 строки внутри блока 7 либо добавить как опциональный 8-й блок.

### Советы по конверсии
1. Первый экран должен ответить на 3 вопроса за 3 секунды: что делаете, для кого, почему доверять именно вам. Личное фото/видео повышает доверие сильнее стоковой графики.
2. CTA размещайте минимум 3 раза: в hero, после кейсов (пик доверия) и в финале.
3. Формулируйте оффер через результат для клиента, а не через технологию: не «делаю сайты на React и ИИ-агентов на GPT», а «увеличиваю конверсию сайта» / «автоматизирую общение с клиентами» — техстек указывайте вторым слоем, в кейсах или услугах.

## 3. Тренды и фишки 2026 года

Изучены awwwards.com (раздел design-trends), css-tricks.com и дополнительный поиск по актуальным трендам веб-дизайна 2025-2026.

### 1. Bento-grid layout с интерактивной глубиной
Модульная сетка из карточек разного размера — доминирующий паттерн (Microsoft, Spotify, YC-стартапы). Карточки не статичны: при наведении расширяются, проигрывают видео или показывают доп. слой данных.
**Почему подходит:** идеально структурирует разноплановый контент портфолио — отдельные «плитки» под сайты, приложения, ИИ-агентов, стек, кейсы — без громоздкой навигации.
**Реализация:** чистый CSS Grid + `grid-template-areas`, hover-transitions на transform/filter — без доп. библиотек.

### 2. «AI-glow» эстетика: тёмная тема + неоновые/диффузные свечения
Мягкие цветные glow-эффекты вокруг кнопок и карточек, градиенты, дуотон, иногда иридесцентные текстуры.
**Почему подходит:** визуально сигнализирует «это ИИ-специалист» без явных баннеров — сама эстетика транслирует специализацию.
**Реализация:** чистый CSS (`box-shadow` с цветным blur, `backdrop-filter`, градиенты), без доп. библиотек.

### 3. Scroll-driven анимации и кинетическая типографика
Элементы проявляются и трансформируются при скролле; заголовки «строятся» по буквам или меняют начертание variable-шрифта. Нативный CSS `animation-timeline: scroll()/view()` в 2026 уже широко поддерживается.
**Почему подходит:** создаёт «премиальное» ощущение продукта малыми средствами — важно для специалиста, продающего впечатление от интерфейсов.
**Реализация:** нативный CSS scroll-timeline для простых случаев; GSAP ScrollTrigger + Lenis (smooth scroll) для более сложной хореографии секций.

### 4. Лёгкие 3D/WebGL-акценты и живой курсор
Точечные интерактивные объекты (вращающийся продукт, частицы, реагирующий на курсор фон) и кастомный курсор с микро-анимацией.
**Почему подходит:** демонстрирует техническую компетентность без перегрузки — один-два акцента, а не весь сайт в 3D.
**Реализация:** React Three Fiber (для React-стека) или Spline (визуальный редактор 3D-сцен, экспорт в код без шейдеров).

### 5. Живое интерактивное демо ИИ-агента прямо на странице
Встроенный мини-чат или виджет, где посетитель может «потрогать» реального ИИ-агента вместо статичных скриншотов в портфолио.
**Почему подходит:** для специалиста по ИИ-агентам — самый сильный аргумент: «не рассказываю, а показываю» прямо на лендинге, резко повышает доверие и конверсию.
**Реализация:** клиентский чат-виджет, дергающий API через тонкий serverless-эндпоинт, с лёгкой анимацией печати текста (CSS/Framer Motion) — без сложной инфраструктуры.

## Общий итог и рекомендации для проекта

- **Стиль:** тёмная тема, неоновый/AI-glow акцент, bento-grid для карточек услуг и кейсов, glassmorphism-элементы.
- **Шрифты:** гротеск для текста (Inter / General Sans / Space Grotesk) + моно-шрифт (JetBrains Mono / Space Mono) для стека технологий и технических деталей.
- **Структура:** 7 блоков — Hero-оффер → Боль/позиционирование → Услуги → Кейсы (главный блок) → Процесс работы → Отзывы/УТП → Финальный CTA/контакты.
- **Ключевые фишки:** один сильный интерактивный акцент в hero (частицы/3D/курсор), scroll-driven анимации между секциями, и — как главное конкурентное преимущество — живое демо ИИ-агента прямо на странице вместо статичного скриншота.

## 4. Экспертная оценка

Три независимых аудита концепции (структура из раздела 2 + тренды из раздела 3) с фактчекингом источников.

### 🎨 Эксперт 1: UX-дизайнер (10 лет опыта)

**Типичные ошибки новичков и риски в концепции**
- Смешение трёх разных аудиторий (сайты / приложения / ИИ-агенты) в одном УТП размывает hero-заголовок — риск присутствует.
- Кейсы без цифр в контексте (метрика без базы) — зона риска при наполнении контентом.
- Один финальный CTA на 7 блоков может быть недостаточен для мультисегментной аудитории — нужны микро-CTA после карточек услуг/кейсов.

**Что обязательно, но не учтено в концепции**
- **Скорость загрузки как метрика проекта.** Рост загрузки с 1 до 2.4 сек снижает конверсию с ~39% до ~1.9%; 53% мобильных пользователей уходят при загрузке дольше 3 сек ([Huckabuy](https://huckabuy.com/20-important-page-speed-bounce-rate-and-conversion-rate-statistics/), [Envisage](https://www.envisagedigital.co.uk/website-load-time-statistics/)).
- **Mobile-first, а не адаптив постфактум.** До 82.9% трафика лендингов — мобильный ([SEOSherpa](https://seosherpa.com/landing-page-statistics/)); bento-grid/3D/курсор нужно проектировать сразу под мобильные.
- **`prefers-reduced-motion`** — обязательное требование доступности для всех scroll-анимаций (вестибулярные нарушения у части аудитории) ([web.dev](https://web.dev/prefers-reduced-motion/)).
- **WCAG-контраст неона на тёмном фоне** — минимум 4.5:1 для текста, 3:1 для крупных элементов; неоновые пары часто проваливают эту проверку «на глаз» ([WebAIM](https://webaim.org/articles/contrast/)).
- Фокус-стейты и ARIA для glassmorphism-карточек и live-чата; fallback для 3D/частиц при отключенном JS и на слабых устройствах.

**Топ-3 риска для проекта**
1. **3D/WebGL-hero без mobile-бюджета** — главный кандидат на мгновенное закрытие вкладки при 80%+ мобильного трафика. Нужен лёгкий 2D/CSS-фолбэк и лимит FPS/треугольников.
2. **Неон без проверки контраста по WCAG AA** — репутационный риск для специалиста, продающего веб-экспертизу.
3. **Scroll-анимации на всех 7 блоках без `prefers-reduced-motion`** — раздражает и бьёт по Core Web Vitals (CLS/INP). Сильные эффекты оставить в 1-2 точках (hero, переход к кейсам), остальное — лёгкий fade/slide.

### 📊 Эксперт 2: Маркетолог, специалист по конверсии (CRO)

**Оффер**
Широкий список услуг в заголовке («сайты, приложения и ИИ-агенты») размывает сообщение — специализация конвертирует лучше. Решение — не убирать услуги, а сменить порядок: сначала результат/боль клиента, форматы услуг — вторым слоем (в подзаголовке или в блоке услуг).
Альтернативные варианты заголовка:
1. «Превращаю сайт, приложение или ИИ-агента в канал продаж — за 2-4 недели, без студийной цены и переписки с менеджерами».
2. «Один эксперт вместо отдела разработки: сайты, приложения, ИИ-агенты, которые окупаются в первый квартал».
3. Текущий вариант («...которые приносят деньги, а не просто выглядят красиво») хорош контрастом с конкурентами, но стоит начинать с боли/результата, а не с перечисления форматов.

**Недостающие триггеры доверия для соло-специалиста**
Ключевой страх клиента — «единая точка отказа» (в отличие от агентства). Явно закрывает это:
- Личность и публичность (фото/видео, живые соцсети/блог, а не формальный «About»).
- **Гарантии и снижение риска** — что будет, если что-то пойдёт не так; поддержка после сдачи. В текущей структуре отсутствует полностью.
- Быстрый канал связи с обещанием срока ответа.
- Верифицируемые кейсы (ссылка на сайт клиента, а не абстрактные проценты).
- **Блок FAQ/возражений** («а если вы заболеете/уедете», «работаете по договору?») — в структуре не предусмотрен, а для соло-мастера это один из главных барьеров.

**Путь клиента и точки отвала**
Наибольший риск — переход Hero → блок боли (решение «остаться» принимается почти мгновенно, размытый оффер = ранний отвал) и финальная форма заявки (каждое лишнее поле снижает конверсию; лид-формы для digital-услуг обычно конвертируют в районе единиц процентов ([First Page Sage](https://firstpagesage.com/seo-blog/landing-page-conversion-rates-by-industry/))). Правило «минимум 3 CTA» подтверждено — это общепринятая практика для длинных one-page лендингов, а не противоречие принципу «один CTA».

**Топ-3 изменения для конверсии**
1. Переписать hero: результат клиента на первом месте, форматы услуг — вторым планом; микро-CTA с низким порогом входа («Посмотреть кейсы») рядом с основным.
2. Добавить блок гарантий/FAQ (8-й блок или врезка перед финальным CTA) — прямая компенсация риска «доверия к одному человеку».
3. Сократить финальную форму до 2-3 полей + продублировать быстрый канал связи (Telegram/WhatsApp) рядом.

### 👤 Эксперт 3: Потенциальный клиент — владелец малого бизнеса

*(от первого лица, реакция на концепцию с телефона, слабый интернет)*

**Первые 5 секунд:** тёмная неоновая эстетика читается как «сайт IT-стартапа/крипто», а не как «человек, который сделает мне сайт». Заголовок про деньги — плюс, воспринимается лучше типовых клише. Термин «ИИ-агент» непонятен без перевода на пользу лично для меня.

**Что удержит:** кейсы с конкретными цифрами похожего бизнеса; живое демо ИИ-агента — но только если оно реально быстро и качественно отвечает, иначе это худший вариант, чем его отсутствие; понятные 4 шага процесса.

**Что оттолкнёт:** отсутствие цен или хотя бы вилки бюджета (без этого красивый дизайн выглядит как воронка для дорогой продажи); перегруз анимациями на слабом телефоне/интернете — прямое противоречие обещанию «делаю быстрые эффективные продукты»; ощущение, что неоновый стиль «не для моего скромного/консервативного бизнеса».

**Из фактчекинга:** исследования подтверждают, что решение о доверии к сайту формируется почти мгновенно (~50 мс, halo-эффект) на основе привычности/чуждости визуального языка — тёмная неоновая эстетика для консервативной аудитории малого бизнеса скорее считывается как «чужая». Трендовые/агрессивные UX-паттерны могут повышать вовлечённость метрически, но снижают доверие у пользователей с невысокой цифровой грамотностью. При этом малый бизнес действительно активно принимает AI (около 68% так или иначе уже используют), но ценит «человеческий голос» и проверяемые факты выше технотерминов.

**3 вещи, которые клиент попросил бы изменить:**
1. Добавить хотя бы ориентировочные цены/диапазоны бюджета.
2. Дать облегчённую/быструю версию для мобильного и слабого интернета.
3. Простое человеческое объяснение «зачем мне ИИ-агент», а не голый buzzword.

## Сводный список рекомендаций (по итогам экспертного аудита)

1. **Пересобрать hero-оффер**: на первом месте — результат/боль клиента, форматы услуг (сайты/приложения/ИИ-агенты) — вторым слоем в подзаголовке или в блоке услуг, а не в заголовке.
2. **Добавить блок гарантий и FAQ/возражений** (8-й блок или врезка перед финальным CTA): что если форс-мажор, поддержка после сдачи, работа по договору — закрывает главный страх клиента перед соло-специалистом.
3. **Указать хотя бы ориентировочные цены/вилку бюджета** — иначе эффектный дизайн без цены читается как манипулятивная воронка (мнение эксперта-клиента подтверждено фактчекингом по доверию).
4. **Спроектировать hero-акцент (3D/частицы/курсор) mobile-first с лёгким fallback**: жёсткий бюджет FPS/полигонов, статическая деградация для слабых устройств — 80%+ трафика лендингов мобильный, а медленная загрузка прямо противоречит позиционированию «быстрый и эффективный».
5. **Проверить контраст неоновых акцентов по WCAG AA** (4.5:1 текст / 3:1 крупные элементы) на этапе макета, а не «на глаз».
6. **Ограничить scroll-driven анимации 1-2 сильными точками** (hero, переход к кейсам) и обязательно уважать `prefers-reduced-motion` — остальные блоки оформить лёгкими fade/slide.
7. **Добавить микро-CTA после карточек услуг и после кейсов**, не только один финальный CTA — снижает риск отвала на мультисегментной аудитории.
8. **Сократить финальную форму заявки до 2-3 полей** + продублировать быстрый канал связи (Telegram/WhatsApp) рядом с формой.
9. **Дать простое объяснение «зачем вам ИИ-агент»** на языке пользы (экономия времени/денег), а не техническим термином без перевода — особенно для нетехнической аудитории малого бизнеса.
10. **Live-демо ИИ-агента в кейсах — только если оно стабильно быстрое**: заложить skeleton-состояние, timeout и закешированные ответы на случай сбоя API, иначе это работает против доверия сильнее, чем его отсутствие.
